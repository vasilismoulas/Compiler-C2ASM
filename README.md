Compiler that converts C (a subset of C's grammar rules) code into AssemblyX86 (without the use of irvine's library).

## File overall operation description

### testlexer.g4
<div style="border-top: 3px dotted #bbb">
<br>
Breaks the source code into smaller pieces called tokens. Each token represents a basic unit of meaning, such as keywords, operators, identifiers, or literals.

<br>

### testparser.g4
<div style="border-top: 3px dotted #bbb">
<br>
Takes the token stream generated by the lexer and checks if the sequence adheres to the grammar rules of the programming language. The parser builds a structured representation of the program (parse tree)

<br>

### Program.cs
<div style="border-top: 3px dotted #bbb">
<br>
Program. cs file is the entry point of the application. It is a C# file that contains a Main method, which is the starting point of the application.

* #### Main funciton
Responsible for executing compiler's [workflow/phases](#archtecture) through the compilation process.

<br>

### TestVisitor.cs
<div style="border-top: 3px dotted #bbb">
<br>

Responsible for the syntax tree creation.

* #### TestVisitor class
Implements ANTLR's generated visitor(testparserBaseVisitor<int>) and overrides the corresponding Visit methods of each grammar rule.

<br>

### ASTElement.cs
<div style="border-top: 3px dotted #bbb">
<br>

* #### ASTElement class
  Blueprint of ASTTree's kinds of nodes(ASTComposite & ASTTerminal).

* #### ASTComposite class
  Blueprint of ASTTree's non-terminal nodes

* #### ASTTerminal class
  Blueprint method of ASTTree's terminal nodes

<br>

### ASTElementConcrete.cs

<div style="border-top: 3px dotted #bbb">
<br>

Contains concrete classes implementing the ASTComposite & ASTTerminal abstract classes.

<br>

### ASTGenerator.cs
<div style="border-top: 3px dotted #bbb">
<br>
Responsible for the abstract syntax tree creation.

#### ASTGenerator class
Implements ANTLR's generated visitor(testparserBaseVisitor<int>) and overrides the corresponding Visit methods of each grammar rule.

<br>
